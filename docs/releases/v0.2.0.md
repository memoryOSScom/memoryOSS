# memoryOSS v0.2.0

memoryOSS v0.2.0 is the first release that reflects the full P9 through P13 surface instead of the older v0.1.1 snapshot. This release is still pre-1.0, but it materially expands the runtime, packaging, compatibility, and ecosystem-default story.

## Highlights

- Shipped the universal memory runtime surface: runtime contract publication, memory passport bundles, history replay, time-machine controls, cross-app adapters, and the public conformance kit.
- Hardened daily-use memory quality: extraction evaluation, contradiction detection, lifecycle automation, identifier-aware reranking, summary and evidence recall, review inbox, and confidence gating.
- Expanded operator and team surfaces: memory HUD, ambient connector mesh, governance, trust and revocation fabric, universal reader, and bootstrap support for managed team rollouts.
- Completed the install and compatibility lane: Windows release artifacts, Docker and GHCR paths, update plane, compatibility and LTS coverage, stronger Claude/Codex setup guarantees, and ecosystem-default setup work.

## Public Proof

- 302 automated checks across unit, integration, TypeScript, wizard, conformance, and soak surfaces.
- 20K memory retention benchmark with early high-signal memories still retrievable as the corpus grows.
- 44.4% token-efficiency improvement in the published repeated-task Claude benchmark.
- 2-hour soak stability plus open benchmark and compatibility lanes.

## Upgrade Notes

- Re-run `memoryoss setup --profile <profile>` on workstations that were configured on older builds so the current managed Claude, Codex, or Cursor surfaces are refreshed.
- Run `memoryoss doctor --repair` after upgrade if a machine has drifted or if you want the selected setup profile replayed deterministically.
- `v0.2.0` is still a public beta release. APIs, config, and packaging details may continue to change before `v1.0.0`.
