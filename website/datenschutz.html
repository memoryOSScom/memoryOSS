2. Overview

This privacy policy covers both the website memoryoss.com and the locally run memoryOSS software / plugin. Different parts apply depending on how you use memoryOSS.

The memoryOSS software / plugin runs locally on your own system and does not send telemetry to us by default. Memories, embeddings, prompts, and configuration data remain on your local system unless you explicitly connect memoryOSS to a third-party upstream model or service.

memoryOSS does not require a memoryOSS account and does not use a hosted memoryOSS backend or sync service for the local software / plugin.

If you use memoryOSS with upstream LLM providers or related external services, selected requests and data may be sent directly from your environment to those providers so the integration can work. In that case, the privacy terms of the respective provider apply in addition to this policy.

The sections below on hosting, CDN, logs, and website access apply to memoryoss.com. The locally run software / plugin is governed by the local processing model described above.

3. How memoryOSS Handles Memory

The points below summarize the product's local-first behavior in plain language. They complement, but do not replace, the legal sections of this privacy policy.

  • Local-first — memory storage, embeddings, and search run on your own machine by default.
  • No telemetry by default — the local software / plugin does not send analytics or usage tracking to us by default.
  • Trust-gated injection — memories are not blindly fed back into prompts; relevance, confidence, and trust checks determine what is eligible for injection.
  • Namespace isolation — memory can be scoped so projects and workspaces stay separated instead of bleeding into each other.
  • Backup, export, and delete — the software includes backup, export, and deletion capabilities for local memory management.
