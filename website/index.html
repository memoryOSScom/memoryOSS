Setup

Three commands. One minute.

Install, run the wizard, start your agent as usual.

1

Install

Single binary. No deps.

cargo install --git https://github.com/memoryOSScom/memoryoss
2

Setup

Wizard detects your tools, writes config, and sets up memory.

memoryoss setup
3

Use

Start Claude or Codex as usual. Memory works automatically.

claude
codex
cursor

If your auth setup changes later — for example from OAuth to API key or back — run memoryoss setup again so the integration path can be updated safely.

Manual proxy mode (optional) — use explicit BASE_URL exports if you want to point API-key clients at the proxy yourself. Codex OAuth stays MCP-first.

# Anthropic export ANTHROPIC_BASE_URL=http://127.0.0.1:8000/proxy/anthropic/v1 # OpenAI export OPENAI_BASE_URL=http://127.0.0.1:8000/proxy/v1 # API-key clients through the local proxy claude codex cursor
Under the Hood

What keeps memory useful.

Relevant Recall

Semantic, exact-match, full-text, and recency signals work together so useful context wins over vague similarity.

Thresholded Injection

Memories must clear relevance and trust thresholds before they consume context window budget.

MCP Built-in

4 tools: store, recall, update, forget. Works with any MCP client.

Quarantine Before Injection

Auto-extracted facts start unverified. Only memories that pass confidence and trust checks are eligible for injection.

Decay and Dedup

Duplicate memories are collapsed, stale facts lose influence, and old noise stops consuming context.

Namespace Isolation

Projects and agents can be kept separate so memory stays scoped instead of leaking across workspaces.

Architecture

How context gets added.

The model keeps its general knowledge. memoryOSS adds relevant project context.

  ├─ Forward   send to upstream LLM
