Roadmap

The runtime core is shipped. The next block is default-install utility.

memoryOSS has finished the `P10` to `P12` runtime build: operator control, adaptive recall, portability, reader, trust fabric, update plane, compatibility discipline, and public utility proof are all in the product now.

Public beta • P10/P11/P12 complete • 20k retention benchmark • 3 everyday utility loops • signed rollback smoke • Claude/Codex compatibility matrix

This page is intentionally short. It is not a release contract. It is the public answer to one question: what is already real, what is next, and what is deliberately not the current bottleneck.

The biggest remaining challenge is no longer inventing another memory primitive. It is making the finished runtime the obvious default install across supported clients, machines, and teams.

Now — shipped

What is already real

  • Portable runtime contract with conformance fixtures, reader, dry-run import, replay preview, and artifact validation.
  • Utility-grade controls with HUD, review queue, policy firewall, trust and revocation, rollback, and compatibility/LTS checks.
  • Public proof loops for retention, long-memory regression, extraction quality, token-efficiency, update rollback, and everyday utility.
Next — post-P12

What comes next

  • Distribution channels that make the supported Claude, Codex, Cursor, and local-team install paths boring and first-class.
  • Ecosystem default surfaces for signer discovery, trust roots, bundle discovery, and zero-friction onboarding across machines.
  • Live proof alignment so website, release artifacts, docs, and onboarding story stay synchronized instead of drifting between deploys.
Later — only with signal

What stays later

  • Deeper research lanes like recursive recall, richer primitive algebra, and coprocessor expansion stay secondary unless they improve real utility.
  • Broader ambient capture only grows further if governance, review, and noise control stay boring under real use.
  • Hosted/cloud expansion remains downstream of proving the local-first product should be installed everywhere first.

What changed

The old public roadmap was stale. It still described `P12` as unfinished even though the shipped build now includes team governance, universal reader, trust fabric, rollback/update plane, compatibility/LTS, and the public everyday-utility loop.

  • The roadmap now treats `P10` through `P12` as complete.
  • The next real block is distribution and ecosystem-default adoption, not another cleanup task hidden inside `P12`.
  • The public benchmark page remains the authoritative proof surface for what the current build actually does.
