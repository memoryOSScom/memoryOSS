Roadmap

Where memoryOSS is going next.

Short, public, and deliberately narrow. This is direction, not a release contract.

memoryOSS is still in beta. The near-term goal is not more integrations or cloud packaging. The goal is to make agent memory smaller, cleaner, more explainable, and more useful over time.

That means lifecycle control, better trust signals, stronger consolidation, and safer injection before broadening the surface area.

Now — v0.1.0

What we're hardening now

  • Lifecycle + trust so weak or superseded memory loses influence instead of lingering forever.
  • Explainability for why memories were recalled or injected.
  • Benchmark and test hardening across wizard, proxy, MCP, and scale paths.
Next — v0.1.5

What we're building next

  • Contradiction detection to mark conflicting memories as contested.
  • Cognitive consolidation that turns repeated facts into smaller, higher-value knowledge.
  • Better proactive injection so relevant memory comes back before the same mistake does.
Later — v0.2.0

What can wait

  • Scoped shared-memory modes with tighter project and team boundaries.
  • Hosted cloud service only after the local product is hardened enough to deserve it.
  • Protocol work only after the product proves the shape of the right abstraction.

What this page is for

This roadmap exists to show direction and current priorities. It is intentionally short so it stays honest and updated.
